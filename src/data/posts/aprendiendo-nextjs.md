# Aprendiendo Next.js: Primeras impresiones

Hace unas semanas decidí profundizar en **Next.js** y la verdad es que estoy muy contento con la decisión.

## ¿Por qué Next.js?

Después de trabajar principalmente con React puro, quería explorar algo que me diera más estructura y herramientas out-of-the-box. Next.js parece ser exactamente eso.

### Lo que más me está gustando:

1. **Server Components**: La idea de poder renderizar componentes en el servidor es genial. Me simplifica mucho el manejo de datos.

2. **File-based routing**: No tener que configurar rutas manualmente es un alivio. Solo creas archivos y ya tienes las rutas.

3. **Optimización automática**: Las imágenes, el código, todo se optimiza automáticamente. Eso es tiempo que no tengo que perder.

## Desafíos

No todo es perfecto, claro. Algunas cosas que me están costando:

- Entender bien cuándo usar Server Components vs Client Components
- La configuración inicial puede ser un poco abrumadora
- Algunos conceptos nuevos como `generateStaticParams` requieren tiempo para asimilarlos

## Próximos pasos

Quiero seguir explorando:
- API Routes
- Middleware
- Optimizaciones avanzadas
- Deployment
